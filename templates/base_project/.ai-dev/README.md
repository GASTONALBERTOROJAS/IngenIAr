# `.ai-dev/` for this project

Use this folder to keep project work traceable from requirements through QA.

## Minimum expectations

- capture new scoped work in `requirements/` when the request becomes a real project task
- add a plan in `plans/` before meaningful implementation
- record important decisions in `decisions/`
- keep `state/current_state.md` current
- add QA evidence in `qa/` before closing meaningful work
- store durable summaries or reports in `artifacts/`

This local `.ai-dev/` tree is the project-level traceability layer for the base IngenIAr workflow.
